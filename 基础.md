### ls  
3. `ls` ==列出目录下有什么==  

`-l` ==长格式显示，包含权限、所有者、大小、时间等==  
```bash    
    root@dell-PowerEdge-R740xd:/home/dell# ls -l  
    总计 44  
    drwxr-xr-x 2 dell dell __4096__ 10月 29 12:19 公共  
    drwxr-xr-x 2 dell dell __4096__ 10月 29 12:19 模板  
```
`-lh` ==人类可读的格式显示文件大小==
```bash
    >root@dell-PowerEdge-R740xd:/home/dell#  ls -lh  
    总计 44K  
    drwxr-xr-x 2 dell dell **4.0K** 10月 29 12:19 公共  
    drwxr-xr-x 2 dell dell __4.0K__ 10月 29 12:19 模板  
```
`-a` ==显示所有文件，包括隐藏文件（以.开头的文件）==
```bash
    >root@dell-PowerEdge-R740xd:/home/dell# ls -a  
    .     桌面               __.gtkrc-2.0__                 __.Xauthority__  
    ..    __.bash_history__  __.ICEauthority__              __.xinitrc__  
```

`-lt` ==按修改时间排序（最新的在前）==  
`-lS` ==按文件大小排序（最大的在前）==  
`-lr` ==反向排序==  ( 搭配 -S / -t )

root@dell-PowerEdge-R740xd:~# cat 1.txt 
你好
你好

### cp

7. cp ==复制文件和目录==

`cp source dest` ==复制源文件到目标位置==
```bash
cp file1.txt file2.txt
```

`cp -i` ==交互式复制，覆盖前提示==
```bash
cp -i file1.txt file2.txt
```

`cp -r` ==递归复制目录及其内容==
```bash
cp -r dir1 dir2
```

`cp -p` ==保留原文件的属性(权限、时间戳等)==
```bash
cp -p file1.txt file3.txt
```

`cp -v` ==显示详细的复制过程==
```bash
cp -v file1.txt file4.txt
```

`cp -u` ==更新模式，仅当源文件比目标文件新时才复制==
```bash
cp -u file1.txt file2.txt
```

`cp -l` ==创建硬链接而非复制文件内容==  [软 / 硬连接说明](#软硬连接说明)
```bash
cp -l file1.txt file_hardlink.txt
```

`cp -s` ==创建符号链接（软链接）==
```bash
cp -s file1.txt file_softlink.txt
```

`cp -f` ==强制复制，覆盖已存在的目标文件而不提示==
```bash
cp -f file1.txt file2.txt
```

`cp -t` ==指定目标目录，源文件放在最后==
```bash
cp -t /target/dir/ file1.txt
```

### ln
8. ln ==创建文件链接==

`ln` ==创建硬链接== #
```bash
ln file1.txt file1_hardlink.txt
```

`ln -s` ==创建符号链接（软链接）==
```bash
ln -s file1.txt file1_softlink.txt
```

`ln -f` ==强制创建链接，覆盖已存在的目标文件==
```bash
ln -f file1.txt existing_link.txt
```

`ln -v` ==显示详细的链接创建过程==
```bash
ln -v file1.txt file1_link.txt
```

`ln -t` ==指定链接文件的目标目录==
```bash
ln -t /home/user/links/ file1.txt file2.txt
```

`ln -P` 直接创建硬链接到符号链接（而不是解引用）
```bash
ln -P symlink_source hardlink_to_symlink
```

`ln -L` 解引用所有遇到的符号链接
```bash
ln -L symlink_source new_hardlink
```
### 软硬连接说明
😵‍💫假设我们有如下文件结构：😵‍💫
```bash
实际文件.txt  -> 内容为 "Hello World"
符号链接_A    -> 指向 ‘实际文件.txt‘  (软链接)
```
`ln -P 符号链接_A 硬链接_B`
操作对象：==符号链接_A 这个文件本身。==

结果：硬链接_B 是 符号链接_A 的硬链接。现在有两个符号链接文件（符号链接_A 和 硬链接_B），它们都存储着相同的路径信息 "实际文件.txt"。如果你删除 实际文件.txt，这两个符号链接都会变成“悬空”的坏链接。

`ln -L 符号链接_A 硬链接_C`

操作对象：符号链接_A 最终指向的 实际文件.txt。

结果：硬链接_C 是 实际文件.txt 的硬链接。现在 实际文件.txt 和 硬链接_C 指向同一个数据块。符号链接_A 只是一个中间人。

`ln -s 符号链接_A 符号链接_D`

操作对象：创建一个新的符号链接。

结果：符号链接_D -> 符号链接_A -> 实际文件.txt

|  | 软连接 | 硬链接 |
|-------|--------|------|
|特性	|硬链接	|软连接|
|本质	|同一个文件的多个名字	|指向文件的快捷方式|
|inode	|与原文件相同	|新的inode|
|磁盘空间	|不额外占用	|占用少量空间存储路径|
|原文件删除	|数据仍存在	|链接失效|
|跨文件系统	|❌ 不支持	|✅ 支持|
|指向目录	|❌ 不支持	|✅ 支持|
|文件类型	|普通文件	|链接文件 (l)|
